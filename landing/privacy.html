Privacy

Privacy Policy

Last updated 29 May 2026 · Applies to System Load v1.0.2 for macOS and this website
The short version

System Load collects nothing. No analytics, no telemetry, no crash reporting, no accounts, no advertising, no fingerprinting. Your CPU, RAM, GPU, network and battery readings are computed on your Mac and never leave the device.

The app makes exactly one kind of network request — an optional, once-a-day check for updates — and you can turn it off. That's the entire policy. Everything below is just the honest detail.

01 What the app reads

Everything System Load shows is read straight from the macOS kernel, on your machine, and held only in memory.

To display its metrics, the app queries the operating system's own statistics interfaces:

  • CPU load — host_statistics(HOST_CPU_LOAD_INFO), from tick deltas
  • RAM & compression — host_statistics64(HOST_VM_INFO64)
  • Memory pressure — sysctl kern.memorystatus_vm_pressure_level
  • Swap — sysctl vm.swapusage
  • Network throughput — getifaddrs per-interface byte counters
  • GPU & battery — the IORegistry
  • Top processes — /bin/ps, sampled only while the popup is open

These are aggregate system numbers — percentages, byte counts, process names already visible in Activity Monitor. They are displayed live and discarded. A short rolling history (about the last two minutes, for the CPU and GPU sparklines) is kept in memory only and is gone the moment you quit. None of it is recorded to disk, and none of it is ever transmitted anywhere.

02 What it never does

To be unambiguous, System Load contains:

  • No analytics or telemetry — no usage stats, no event tracking, no “anonymous diagnostics”
  • No crash reporting service — nothing phones home after a crash
  • No advertising or marketing SDKs
  • No accounts, logins, or sign-in — there is nothing to register
  • No fingerprinting or unique identifiers
  • No data brokers or third-party data sharing — there is no data to share

03 The one network request: update checks

The metrics engine makes zero network calls. The only component in the entire app that touches the internet is the auto-update framework, Sparkle. When automatic checks are enabled, about once a day it fetches a small update manifest to see whether a newer version exists.

What it requests
https://raw.githubusercontent.com/kyzdes/system-load/main/appcast.xml — a public file
How often
Roughly every 24 hours, and when you choose “Check for Updates…” manually
What the server sees
The unavoidable basics of any HTTPS request: your IP address and a standard User-Agent (which includes the app version, macOS version and CPU architecture). This goes to GitHub, which hosts the file.
What is NOT sent
Sparkle's optional anonymous system-profile feature is disabled — no hardware survey or usage profile is transmitted. No metrics, no identifiers, no personal data.
Integrity
Updates are verified with an EdDSA signature before they're allowed to install
Turn it off
Menu → Settings… → Software Update → disable automatic checks. With it off, the app makes no network requests at all until you check manually.

If an update is found and you accept it, the new version is downloaded from the project's GitHub Releases. That download, like any download, is handled by GitHub.

04 What's stored on your Mac

The app keeps a few preferences locally, and nothing else:

Preferences
Refresh interval, which menu-bar items to show, glyphs-vs-text, and update settings — stored in macOS UserDefaults for this app
Launch at login
If you enable it, the choice is registered with the system via SMAppService (also editable in System Settings → General → Login Items)
Cloud sync
None. Nothing is written to iCloud or any server

Removing the app removes the app. Preferences are standard local app settings on your own machine.

05 Permissions & privileges

System Load runs as a plain menu-bar agent (LSUIElement) — no Dock icon, no main window. It asks for no elevated privileges, no administrator password, no Full Disk Access, and no special entitlements. It reads only the public, aggregate system statistics listed in §01. There is nothing it can see about your files, messages, browsing, or keystrokes — because it never asks for access to any of that.

06 This website

The page you're reading is a static site (hosted on GitHub Pages). It sets no cookies, runs no analytics and embeds no trackers or ad scripts. It loads two third-party resources for presentation only — the React runtime and web fonts — from public CDNs, which receive the standard request metadata (IP, User-Agent) inherent to loading any file.

As with any website, the host (GitHub) may keep standard server access logs; see GitHub's privacy practices below. On a phone, the “Copy link / Email it to myself” button uses your device's own clipboard or mail app locally — nothing is sent to us.

If this site is later moved to a custom domain, this section will be updated to name the host. No analytics will be added.

07 Third parties

System Load has no business relationships that involve your data. The only third parties in the picture are infrastructure:

GitHub
Hosts the source code, the update manifest, release downloads and this website. Subject to GitHub's Privacy Statement
Sparkle
The open-source update framework that runs locally inside the app (see §03)

08 Children's privacy

System Load is a developer/power-user utility and is not directed at children. In any case it collects no personal information from anyone, regardless of age.

09 Changes to this policy

If this policy changes, the “Last updated” date at the top will change with it, and the full history is visible in the project's git history. Material changes (for example, if the app ever began making a new kind of network request — it won't without saying so) would be called out, not slipped in.

10 Contact
